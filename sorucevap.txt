# KINDERLAb ROBOTİCS PROJESİ - KAPSAMLI SORU-CEVAP REHBERİ

## 🚀 PROJE GENEL BAKIŞ

### S1: Bu proje nedir ve ne amaçla geliştirildi?

- **E-ticaret:** KIBO robotik kitlerinin satışı
- **İçerik Yönetimi:** Blog, basın bültenleri, medya kapsamı yönetimi
- **Eğitim Platformu:** Kurslar, etkinlikler, öğretim materyalleri

## 🏗️ TEKNİK MİMARİ VE TEKNOLOJILER

### S3: Projenin teknik mimarisi nasıl?
**C:** Proje modern **Full-Stack** mimarisine sahip:
```
Frontend (React/Vite) ↔ Backend (Node.js/Express) ↔ Database (SQLite)
```

### S4: Frontend'de hangi teknolojileri kullandık ve neden?
**C:** 
- **React 18 + TypeScript:** Bileşen tabanlı, tip güvenli geliştirme
- **Vite:** Hızlı geliştirme ve build süreci (Webpack'ten 10x hızlı)
- **React Router:** SPA routing yönetimi
- **TanStack Query:** Server state yönetimi ve caching
- **Tailwind CSS:** Utility-first CSS framework
- **Shadcn/UI:** Hazır, erişilebilir UI bileşenleri
- **Radix UI:** Headless UI primitives
- **FontAwesome:** İkon kütüphanesi
- **TinyMCE:** Rich text editor

**Neden bu teknolojiler?**
- **Performans:** Vite ve React 18'in concurrent features
- **Geliştirici Deneyimi:** TypeScript ile tip güvenliği
- **Hızlı Geliştirme:** Tailwind + Shadcn ile hızlı UI
- **Erişilebilirlik:** Radix UI'ın built-in a11y desteği

### S5: Backend'de hangi teknolojileri kullandık?
**C:**
- **Node.js:** JavaScript runtime
- **Express.js:** Web framework
- **SQLite3:** Dosya tabanlı veritabanı
- **Multer:** Dosya yükleme middleware
- **UUID:** Benzersiz ID üretimi
- **CORS:** Cross-origin resource sharing

**Neden bu seçimler?**
- **Tek Dil:** JavaScript hem frontend hem backend
- **Basitlik:** SQLite kurulum gerektirmez
- **Hız:** Express minimal ve hızlı
- **Güvenlik:** CORS ve dosya filtreleme

### S6: Veritabanı şeması nasıl tasarlandı?
**C:** SQLite veritabanında 9 ana tablo:
```sql
users              # Kullanıcı bilgileri
orders              # Siparişler
order_items         # Sipariş detayları
course_registrations # Kurs kayıtları
contacts            # İletişim formları
blog_posts          # Blog yazıları
press_releases      # Basın bültenleri
events              # Etkinlikler
media_coverage      # Medya kapsamı
```

**Tasarım Prensipleri:**
- **Normalizasyon:** İlişkisel tablolar (orders ↔ order_items)
- **JSON Depolama:** Karmaşık veriler JSON string olarak
- **İndeksleme:** Performans için kritik alanlarda index
- **Constraint'ler:** Veri bütünlüğü için CHECK constraints

### S40: Database migration sistemi nasıl çalışıyor?
**C:** Automated migration system:

**Migration Process:**
```javascript
async runMigrations() {
  await this.addImagesColumnToBlogPosts();
  await this.addGoogleMapsLinkColumnToEvents();
}

async addImagesColumnToBlogPosts() {
  const columnInfo = await this.all(`PRAGMA table_info(blog_posts)`);
  const hasImagesColumn = columnInfo.some(col => col.name === 'images');
  
  if (!hasImagesColumn) {
    await this.run(`
      ALTER TABLE blog_posts
      ADD COLUMN images TEXT NOT NULL DEFAULT '[]'
    `);
  }
}
```

**Benefits:**
- Automatic schema updates
- Backward compatibility
- Data integrity preservation
- Version control for database

## 📚 EK BİLGİLENDİRME - DETAYLI BACKEND ANALİZİ

### Backend Mimarisi ve Veri Akışı - Derinlemesine İnceleme

**Bu bölüm, projenin backend kısmının nasıl çalıştığı, API'lerin ne iş yaptığı ve frontend-backend-veritabanı bağlantılarının nasıl işlediği hakkında detaylı bilgiler içerir.**

#### 🔧 Backend'in Çalışma Prensibi

**Server.js Dosyası (Ana Sunucu):**
- **Port:** 3001'de çalışır
- **Middleware'ler:** CORS, express.json(), multer (dosya yükleme)
- **Static Files:** `/postimages` rotası ile görsel dosyaları serve eder
- **Database Connection:** Başlangıçta otomatik veritabanı bağlantısı ve şema çalıştırma

**Dosya Yükleme Sistemi (Multer):**
```javascript
// Dinamik klasör yapısı
/public/postimages/
├── blog/[postId]/images/     # Blog görselleri
├── press/[pressId]/images/   # Basın bülteni görselleri  
├── media/[mediaId]/images/   # Medya kapsamı görselleri
├── events/[eventId]/images/  # Etkinlik görselleri
└── temp/images/              # Geçici yüklemeler
```

**Güvenlik Filtreleri:**
- Sadece görsel dosyaları (.jpg, .jpeg, .png, .gif, .webp)
- 5MB maksimum dosya boyutu
- Benzersiz dosya adları (timestamp + random number)

#### 🗄️ Veritabanı Mimarisi ve İşleyişi

**SQLite Database Class:**
```javascript
class Database {
    connect()     // Veritabanı bağlantısı
    runSchema()   // Şema dosyasını çalıştır
    runMigrations() // Otomatik güncellemeler
    run()         // INSERT, UPDATE, DELETE
    get()         // Tek satır SELECT
    all()         // Çoklu satır SELECT
}
```

**Otomatik Migration Sistemi:**
- `addImagesColumnToBlogPosts()`: Blog tablosuna images kolonu ekler
- `addGoogleMapsLinkColumnToEvents()`: Events tablosuna Google Maps link kolonu ekler
- Backward compatibility sağlar
- Veri bütünlüğünü korur

**2. Backend'de İstek İşleme:**
```javascript
app.post('/api/blog-posts', async (req, res) => {
    // 1. Veri validasyonu
    const { title, content, excerpt, author, status } = req.body;
    
    // 2. Yeni post objesi oluştur
    const newPost = {
        id: uuidv4(),
        title, content, excerpt, author, status,
        publishDate: new Date().toISOString(),
        images: '[]',
        createdAt: new Date().toISOString(),
        updatedAt: new Date().toISOString()
    };
    
    // 3. Veritabanına kaydet

    // 4. Response döndür
    res.json(newPost);
});
```

**3. Database İşlemi:**
```javascript
async createBlogPost(post) {
    const sql = `
        INSERT INTO blog_posts 
        (id, title, content, excerpt, author, publish_date, status, images, created_at, updated_at)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
    `;
    return this.run(sql, [post.id, post.title, ...]);
}
```

#### 🛡️ Güvenlik ve Hata Yönetimi

**SQL Injection Koruması:**
- Tüm sorgularda parametreli queries kullanılır
- `this.run(sql, [param1, param2])` formatı

**Hata Yakalama Stratejisi:**
```javascript
try {
    const result = await database.someOperation();
    res.json(result);
} catch (error) {
    console.error('Database error:', error);
    res.status(500).json({ error: 'Internal server error' });
}
```
